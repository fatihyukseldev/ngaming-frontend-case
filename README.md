# NGAMING Case Study

A React application that renders the betting bulletin provided by the Nesine case-study API and allows users to add odds to a betting slip.

## Live Demo

https://ngaming-frontend-case.onrender.com/

## Setup

Node.js 22.15 or newer is required.

```bash
npm install
npm run dev
```

The development server runs at `http://localhost:3000`.

Bulletin data is loaded from:

```text
https://nesine-case-study.onrender.com/bets
```

## Project Structure

The application is organized around two features.

### Bulletin

- Loads and validates the API response
- Maps markets and outcomes to bulletin columns
- Handles loading, error, retry, and empty states
- Renders the virtualized event table

### Bet Slip

- Adds, replaces, and removes selections
- Clears the complete coupon
- Calculates the combined odd
- Provides desktop and mobile coupon layouts

## State Management

The Redux store is configured with Redux Toolkit and contains two slices: `bulletin` and `betSlip`.

Selections are stored by event ID:

```ts
type BetSlipState = {
  selectionsByEventId: Record<string, BetSelection>;
};
```

This structure enforces one active selection per event. The selection list and combined odd are calculated through selectors instead of being stored separately.

## Rendering

Bulletin rows are rendered with TanStack Virtual using a fixed row height and a small overscan value. Only the rows around the visible area are mounted in the DOM.

Each `BulletinRow` subscribes only to the selection of its own event. Selecting an odd updates the related event row and the betting slip without requiring every bulletin row to render again.

The bulletin header remains visible while the table is scrolled.

## Bet Slip Rules

- An event can have only one active selection.
- Selecting another outcome from the same event replaces the previous one.
- Selecting the active outcome again removes it.
- Selections from different events can exist together.
- Selections can be removed individually.
- The complete coupon can be cleared.
- Combined odd is calculated from active selections.

On mobile, the coupon is opened from a fixed summary bar at the bottom of the screen.

## Missing Outcomes

Missing or invalid odds are rendered as empty, non-interactive cells. Values that do not exist in the API are not generated by the application.
